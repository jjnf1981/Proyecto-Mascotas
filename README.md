# Descripción de los programas
Este proyecto compara dos programas que resuelven el mismo problema: registrar y mostrar información de mascotas, pero de dos formas diferentes:

- Programación Tradicional: ("tradicional.py"): Usa funciones y variables sin clases. Los datos se ingresan por teclado y se muestran con funciones independientes.
- Programación Orientada a Objetos: ("mascota.py" y "main.py"): Define una clase "Mascota" con atributos (nombre, especie, edad) y métodos ("mostrar_informacion", "hacer_sonido"). Se crean dos objetos y se ejecutan sus métodos.

# Reflexión sobre las diferencias
La diferencia que observo entre ambos paradigmas es la forma de organizar el código. En la programación tradicional, los datos y las funciones están separados: las funciones reciben los datos como parámetros y devuelven resultados. Esto funciona bien para programas pequeños, pero cuando el programa crece, puede volverse difícil de mantener porque cualquier cambio en los datos puede afectar muchas funciones.

En cambio, la Programación Orientada a Objetos agrupa los datos (atributos) y las acciones (métodos) dentro de un mismo objeto. Esto permite que cada objeto sea autónomo y reutilizable. Además, el encapsulamiento facilita el mantenimiento, ya que los cambios en los atributos o métodos se hacen solo dentro de la clase y no afectan al resto del programa.

En resumen, la programación tradicional es útil para tareas simples, pero la POO ofrece una mejor organización, reutilización y escalabilidad para proyectos más complejos.
